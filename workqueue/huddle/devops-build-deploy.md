# DevOps 빌드배포시 고려사항

## 메타
- **유형**: `huddle`
- **작성일**: 2025-12-18

## Why (목적)
로컬 개발 편의성과 클라우드 배포 확장성은 트레이드오프 관계. 팀원들이 이 차이를 인지해야 나중에 재설계 비용 발생 안함.

## What (무엇)
### 개발 vs 프로덕션 차이점
- 개발: 빠른 이터레이션, 파일시스템 직접 접근 OK
- 프로덕션: 스케일아웃 고려, 상태 분리 필수

### 주요 변경 필요 영역
- 파일시스템 직접 사용 → 클라우드 스토리지로 재설계
- 로컬 DB → 별도 격리된 컨테이너 연결 구조

## Why This Way (선택 이유)
- 파일시스템 직접 접근은 단일 인스턴스에서만 동작
- 컨테이너 스케일아웃시 상태 공유 불가
- 클라우드 스토리지/DB 분리가 수평 확장의 전제 조건
