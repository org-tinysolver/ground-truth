# 사람을 위한 문서

팀원들이 읽고 이해하기 위한 문서입니다.

## 목적

이 섹션의 문서는 **사람이 읽고 이해**하기 위해 작성됩니다.

## 문서 영역

| 영역 | 설명 |
|------|------|
| [프로덕트](./products/overview) | 서비스별 온보딩 가이드 |
| [빌더스](./builders/overview) | 혼자서 끝까지 만드는 문화 |
| [DevOps](./devops/overview) | 인프라 & 운영 가이드 |

## 문서 특징

- 맥락과 "왜"에 집중
- 예시와 시각 자료
- 친근한 설명
- 체크리스트로 진행 상황 추적

:::tip 어디서 시작할까?

- **새 프로젝트**: 빌더스 가이드부터
- **인프라 필요**: DevOps 가이드부터
