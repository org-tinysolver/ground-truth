# MVP 가이드

Minimum Viable Product - 실제 사용자와 함께 성장하기

## MVP란?

> "고객이 쓸 수 있는 가장 작은 제품"

POC에서 가설이 검증되면, 실제 사용자가 사용할 수 있는 최소한의 제품을 만듭니다.

## POC vs MVP

```mermaid
flowchart LR
    subgraph POC
        P1[가설 검증]
        P2[기술 가능성]
        P3[내부 테스트]
    end

    subgraph MVP
        M1[실사용 가능]
        M2[사용자 피드백]
        M3[빠른 이터레이션]
    end

    POC --> MVP
```

| 구분 | POC | MVP |
|------|-----|-----|
| 목적 | 기술 검증 | 시장 검증 |
| 사용자 | 내부 | 실제 사용자 |
| 품질 | 실험적 | 사용 가능 |
| 기간 | 1-2주 | 2-4주 |

## MVP 핵심 원칙

<!-- TODO: 상세 원칙 -->

1. **최소 기능만** - 핵심 가치 하나에 집중
2. **빠른 출시** - 완벽함보다 속도
3. **피드백 루프** - 출시 → 피드백 → 개선
