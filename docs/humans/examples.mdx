## 📂 아코디언

<Accordion title="IAM이란 무엇인가요?" defaultOpen={true}>
  Identity and Access Management의 약자로, 클라우드 리소스에 대한 접근 권한을 관리하는 서비스입니다.
  
  - **사용자(User)**: 개별 계정
  - **그룹(Group)**: 사용자 모음
  - **역할(Role)**: 임시 권한 부여
  - **정책(Policy)**: 권한 정의
</Accordion>

<Accordion title="MFA는 왜 필요한가요?">
  Multi-Factor Authentication은 보안을 강화하는 필수 요소입니다.
  
  1. 비밀번호가 유출되어도 추가 인증 필요
  2. 규정 준수 요구사항
  3. 중요 리소스 보호
</Accordion>

<Accordion title="CLI 설정 방법">
  각 클라우드별 CLI 설정 방법입니다:
  
```bash
# AWS
aws configure

# GCP  
gcloud init

# Azure
az login
```
</Accordion>

### 플로우차트

```mermaid
flowchart TD
    A[시작] --> B{로그인 성공?}
    B -->|Yes| C[대시보드]
    B -->|No| D[에러 메시지]
    D --> A
    C --> E{권한 확인}
    E -->|있음| F[리소스 접근]
    E -->|없음| G[권한 요청]
    G --> H[승인 대기]
    H --> E
```

### 시퀀스 다이어그램

```mermaid
sequenceDiagram
    actor User
    participant Console as Cloud Console
    participant IAM as IAM Service
    participant Resource as 리소스

    User->>Console: 로그인 요청
    Console->>IAM: 인증 확인
    IAM-->>Console: 토큰 발급
    Console-->>User: 로그인 성공
    User->>Resource: 리소스 접근
    Resource->>IAM: 권한 확인
    IAM-->>Resource: 권한 승인
    Resource-->>User: 접근 허용
```

### 상태 다이어그램

```mermaid
stateDiagram-v2
    [*] --> 신청중
    신청중 --> 검토중: 신청 제출
    검토중 --> 승인됨: 승인
    검토중 --> 반려됨: 반려
    반려됨 --> 신청중: 재신청

    활성화 --> [*]
```
