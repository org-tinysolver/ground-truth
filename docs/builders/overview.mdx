# 빌더스 오버뷰

우리는 **각자가 오너십을 가지고, 혼자서 버티컬 슬라이스로 제품을 만들어가는 문화**를 지향합니다.

## 핵심 가치

```mermaid
mindmap
  root((빌더스))
    오너십
      내 제품이라는 마인드
      끝까지 책임지기
    버티컬 슬라이스
      기획부터 배포까지
      풀스택 역량
    성장 단계
      POC
      MVP
      Production
    오픈 확장
      내부 오픈소스
      기여 환영
```

## 버티컬 슬라이스란?

```mermaid
flowchart TB
    subgraph Traditional["전통적 방식 (수평)"]
        direction LR
        T1[기획] --> T2[디자인] --> T3[백엔드] --> T4[프론트] --> T5[QA] --> T6[배포]
    end

    subgraph Vertical["버티컬 슬라이스"]
        direction TB
        V1[기능 A<br/>기획~배포]
        V2[기능 B<br/>기획~배포]
        V3[기능 C<br/>기획~배포]
    end
```

**버티컬 슬라이스 = 하나의 기능을 처음부터 끝까지 혼자서**

- ✅ 기획/설계
- ✅ 백엔드 개발
- ✅ 프론트엔드 개발
- ✅ 테스트
- ✅ 배포
- ✅ 모니터링

## 오픈소스 스타일 기여

만들어진 제품은 **내부 오픈소스**처럼 운영됩니다.

```mermaid
flowchart LR
    A[아이디어] --> B[Fork/Branch]
    B --> C[개발]
    C --> D[PR 생성]
    D --> E[코드 리뷰]
    E --> F[머지]
    F --> G[배포]
```
