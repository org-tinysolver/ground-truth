# AI PM

제품 기획, 우선순위 결정, 스펙 작성을 담당하는 AI 팀.

:::info 🎯 집중 영역
**명확한 스펙으로 Dev 블로커 제거**

Dev가 "무엇을 만들어야 하는지" 고민하지 않도록.
:::

## 역할

```mermaid
mindmap
  root((AI PM))
    제품 방향
      비전 정립
      로드맵 관리
    우선순위
      백로그 관리
      임팩트 분석
    스펙 작성
      PRD 작성
      유저 스토리
    커뮤니케이션
      Slack 리포트
      팀 조율
```

## 마이크로 사이클

### 사이클 구조

```mermaid
flowchart LR
    A[문제 정의] --> B[가설 수립]
    B --> C[최소 스펙]
    C --> D[Slack 리포트]
    D --> E{승인?}
    E -->|Yes| F[Dev 전달]
    E -->|No| G[피드백 반영]
    G --> C
```

### 사이클 예시

| 단계 | 시간 | 산출물 |
|------|------|--------|
| 문제 정의 | 30분 | 1문장 문제 정의 |
| 가설 수립 | 30분 | 검증 가능한 가설 1개 |
| 최소 스펙 | 1시간 | 마이크로 PRD |
| 리포트 & 피드백 | - | Slack 스레드 |

**총 사이클: 2시간 이내**

## 위임 구조

```mermaid
flowchart TB
    PM_LEAD[PM Lead]

    subgraph SubAgents["서브 에이전트"]
        SPEC[Spec Writer<br/>상세 스펙 작성]
        PRIORITY[Priority Analyzer<br/>우선순위 분석]
        USER[User Research<br/>유저 리서치]
    end

    PM_LEAD --> SPEC
    PM_LEAD --> PRIORITY
    PM_LEAD --> USER

    SPEC --> PM_LEAD
    PRIORITY --> PM_LEAD
    USER --> PM_LEAD
```

### 위임 규칙

1. **위임 가능**: 상세 스펙 작성, 데이터 분석, 리서치
2. **위임 불가**: 최종 우선순위 결정, 승인 요청

```markdown
### 컨텍스트
[왜 이 작업이 필요한지]

### 제안
[구체적인 제안 내용]

### 검증 기준
[어떻게 성공을 측정할지]

- [ ] 승인
- [ ] 피드백 필요

```

## 산출물

### 마이크로 PRD

```markdown
# [기능명]

## 문제
[1-2문장으로 문제 정의]

## 가설
[검증하고자 하는 가설]

## 범위
- ✅ 포함: [최소 범위]
- ❌ 제외: [명시적 제외]

## 성공 기준
[측정 가능한 기준]

## 다음 단계
[이 사이클 이후 예상되는 다음 스텝]
```
