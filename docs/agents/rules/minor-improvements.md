# 마이너 개선점 처리 기준

> PM이 자율적으로 처리할 수 있는 마이너 개선점의 기준

## 자율 처리 가능 (HITL 불필요)

### 문서화 개선
- 교훈(lessons-learned.md) 추가
- 규칙 문서에 명확화 섹션 추가
- 가이드라인/체크리스트 추가
- 용어 통일, 오타 수정

### 프로세스 명확화
- 기존 규칙에 예시 추가
- 암묵적 규칙의 명문화
- 엣지 케이스 처리 방법 문서화

### 일관성 개선
- 파일 형식 통일
- 네이밍 컨벤션 적용
- 경로 표기 통일

---

## 사용자 승인 필요

### 새로운 규칙/정책 도입
- 기존에 없던 프로세스 추가
- 워크플로우 변경
- 팀 간 상호작용 방식 변경

### 구조 변경
- 폴더 구조 변경
- 파일 이동/삭제
- 스키마 변경

### 판단이 필요한 경우
- 여러 대안 중 선택이 필요
- 트레이드오프가 존재
- 다른 시스템에 영향

---

## 처리 절차

```
문제 발견
    │
    ├─ 자율 처리 범위인가?
    │   ├─ YES → 바로 수정, PR 생성, 보고서 작성
    │   └─ NO → 사용자에게 확인 후 진행
    │
    └─ 완료 후
        ├─ lessons-learned.md에 교훈 추가 (해당 시)
        └─ reports/에 보고서 작성
```

---

```markdown
- [무엇을 수정했는가]

- [왜 자율 처리 범위로 판단했는가]

- [PR 링크]

## 비고
- [추가 사항]
```
