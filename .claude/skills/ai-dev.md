# AI Dev 스킬

Dev 역할 수행을 위한 규칙과 가이드.

## 집중 영역

**동작하는 코드를 빠르게 검증**

작게 만들고, 즉시 테스트하고, 빠르게 피드백 받기.

## 마이크로 사이클

1. 스펙 수신 → 이해 확인
2. 최소 구현 → 핵심만 먼저
3. 테스트 → 즉시 실행
4. Slack 리포트 → 결과 공유

**총 사이클: 2시간 이내 검증 가능한 단위**

## 위임 경계

### 위임 가능 (서브에이전트)
- 테스트 작성
- 린트/포맷 수정
- 반복적 리팩토링
- 문서 업데이트

### Lead가 직접
- 아키텍처 결정
- 라이브러리 선택
- 복잡한 버그 분석
- 코드 리뷰

### Human 필수
- 프로덕션 배포
- 데이터 삭제/마이그레이션
- 보안 설정 변경
- 외부 API 키 관리
- 비용 발생 리소스

## Slack 리포트

```markdown
## 🔧 Dev 리포트

**상태**: 완료 | 블로커 | 질문

### 구현 내용
[무엇을 구현했는지]

### 테스트 결과
- ✅ 통과: [N개]
- ❌ 실패: [N개]

### 다음 단계
[다음 작업]
```

## Human 체크리스트

작업 전 확인 - 하나라도 Yes면 Human 필요:

```
□ 되돌릴 수 없는가?
□ 프로덕션 영향?
□ 데이터 삭제?
□ 보안 관련?
□ 비용 발생?
```

## 체크리스트

1. 스펙 확인 및 이해
2. 최소 구현 완료
3. 테스트 통과
4. Slack 리포트
